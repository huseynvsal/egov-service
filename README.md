# EgovService

A Laravel 12 REST API that acts as an integration proxy between the **InvestAZ** mobile app and Azerbaijan's **AsanFinance** government API. It fetches citizen identity, residence permit, and employment data by FIN (personal identification number), caches results locally, and normalises raw government data into a consistent application format.

---

## Stack

| Layer | Technology |
|---|---|
| Framework | Laravel 12 / PHP 8.4 |
| Auth | Static API key (`X-Api-Key` header) |
| Database | MySQL 8 (external — not in Docker) |
| Cache / Queue | Redis 7 |
| AI | Laravel AI (Anthropic Claude Haiku — country code lookup) |
| Email | AWS SES |
| Error tracking | Sentry |
| Containerisation | Docker — app, nginx, redis, worker |

---

## Requirements

- Docker & Docker Compose
- An external MySQL 8 database
- Credentials for AsanFinance API, Anthropic API, and AWS SES

## Local setup

```bash
# 1. Clone and enter the repo
git clone https://github.com/huseynvsal/egov-service.git
cd egov-service

# 2. Copy env file and fill in credentials
cp .env.example .env

# 3. Install dependencies, build containers, generate app key
make setup
```

`make setup` does not run migrations — the app connects to an existing database.

| `LOW_BALANCE_THRESHOLD` | AsanFinance balance alert threshold (default `500`) |

| `SENTRY_LARAVEL_DSN` | Sentry DSN for error tracking |
| `REDIS_HOST` | Redis host (use `redis` inside Docker) |

## API

All endpoints except `/report` require the header:
```
X-Api-Key: {API_KEY}
```

Missing or invalid key returns `401`:
```json
{ "code": 401, "message": "Unauthorized", "data": null }
```

Every response follows the same envelope:
```json
{ "code": 200, "message": "Success", "data": { } }
```

---

### POST `/api/personal-info`
Fetch citizen identity data by FIN.

**Request:**
```json
{ "fin": "1234567", "docNumber": "AA1234567" }
```

| Field | Required | Rule |
|---|---|---|
| `fin` | Yes | Exactly 7 alphanumeric characters |
| `docNumber` | No | Alphanumeric |

**Response `data`:**
```json
{
  "raw": { "PIN": "1234567", "Name": "VÜSAL", "Surname": "HÜSEYNOV", ... },
  "formatData": {
    "clientName": "Vüsal Rauf oğlu Hüseynov",
    "name": "Vüsal",
    "lastname": "Hüseynov",
    "patronymic": "rauf",
    "clientBirthDate": "1990-01-01",
    "clientBirthCountry": "31",
    "clientPassportFin": "1234567",
    "clientPassportSerialNumber": "AA1234567",
    "clientGender": "1",
    "clientCountry": "31",
    "clientMarital": "2",
    "clientNationality": "31",
    ...
  }
}
```

---

### POST `/api/residence-info`
Fetch residence permit data for non-citizens.

**Request:**
```json
{ "fin": "12345" }
```

| Field | Required | Rule |
|---|---|---|
| `fin` | Yes | 5–7 alphanumeric characters |

**Response `data`:** same `raw` / `formatData` structure as personal-info, with an additional `documentType` field (`"DYI"` = permanent, `"MYI"` = temporary).

---

### POST `/api/employee-info`
Fetch employment history.

**Request:**
```json
{ "fin": "1234567" }
```

| Field | Required | Rule |
|---|---|---|
| `fin` | Yes | Exactly 7 alphanumeric characters |

**Response `data`:**
```json
{
  "Active": [ { "Contract": { ... }, "Employee": { ... }, "Employer": { ... } } ],
  "Deactive": [ ... ]
}
```

---

### GET `/api/balance`
Get current AsanFinance account balance.

**Response `data`:**
```json
[{ "Balance": 1250.50, "Currency": "AZN", "Date": "2024-01-15" }]
```

---

### GET `/report` *(no auth)*
Returns an HTML page with monthly request statistics (personal + employment request counts grouped by year and month).

## Error codes

| Code | Meaning |
|---|---|
| `400` | Unrecognised AsanFinance error |
| `401` | Invalid or missing API key |
| `404` | No record found for FIN |
| `422` | Validation error (request or FIN format) |
| `450` | Citizen has blocked data sharing — must activate in MyGov |
| `500` | Unexpected application error |
| `502` | AsanFinance service or external error |
| `503` | AsanFinance unreachable (connection timeout) |

---

## Caching

Responses from AsanFinance are cached in MySQL. A record is considered stale and re-fetched when either:
- `updated_at` is older than `UPDATE_IDENTITY_AFTER_DAY` days, or
- The document's `ExpireDate` has passed

On re-fetch the old record is **soft-deleted** (not overwritten), preserving a full history of every data change per FIN.

A Redis hot cache sits in front of MySQL with a 6-hour TTL to avoid repeated DB reads for the same FIN within a session.

## Development mode

When `APP_DEBUG=true` **and** `APP_ENV=development`, the `/api/personal-info` and `/api/employee-info` endpoints return Faker-generated mock data instead of calling AsanFinance. No API credentials are required in this mode.

## Scheduled tasks

A cron job on the host runs the Laravel scheduler every minute:
```cron
* * * * * docker exec egov_app php artisan schedule:run >> /dev/null 2>&1
```

The scheduler runs `notify:balance` hourly — if the AsanFinance account balance drops below `LOW_BALANCE_THRESHOLD`, an alert email is sent to all addresses in `LOW_BALANCE_NOTIFY_EMAILS`.
